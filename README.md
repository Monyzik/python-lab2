# Лабораторная работа №2

## Как использовать:

В папке ~/python-lab2 можно прописать:

```commandline
python -m src.main
```

Чтобы завершить работу командной строки можно прописать ``exit`` или ``^D``

Изначально текущая директория: '~'
## Реализованные команды:
### cd:
```
cd <path>
```
При отсутствии path перемещает в '~'
### ls:
```
ls [<path1>, <path2>, ...] -[l]
```
Может выводить информацию из нескольких директорий,
но она никак не разделяется, т.е выводится одним массивом данных
### cat:
```
cat <file>
```
### cp
```
cp <from_path> <to_path> -[r]
```
Без флага -r не копирует директорию, даже если она пуста,
при его наличии копирует как директории, так и файлы
### mv
```
mv <from_path> <to_path>
```
### rm
```
rm <path> -[r]
```
Без флага -r не удаляет директорию, даже если она пуста,
при его наличии удаляет как директории, так и файлы.
Блокируется удаление директорий: './' '../' и '/'.
Но пользователь может прописать полный путь до директорий './', '../'
и удалить их, если это была директория '../', то текущая директория смениться на '~'
### zip/tar
```
zip <path> <archive_path>
```
```
tar <path> <archive_path>
```
### unzip/untar
```
unzip <archive_path>
```
```
untar <archive_path>
```
### grep
```
grep <pattern> <path> -[ir]
```
Без флага -r не ищет в директории,
при его наличии может искать как в директории, так и в файлах
### history
```
history <N>
```
Если N не задано, то выводит всю историю в противном случае выводит
max(N, max_n), где max_n - количество сохраненных команд


### Некоторые особенности работы:
- Считываются все дополнительные параметры,
если имеются дублирующиеся или неизвестные параметры, то они просто игнорируются
- Дополнительные параметры никак не зависят от их места в написанной команде, т.е
они могут располагаться как сразу после основной команды, так и после всех аргументов
- При выполнении undo команды не исчезают из истории, но undo действительно отменяет последнюю
команду и при повторном вызове отменит уже следующую команду
- Пробелы в файлах должны выделяется, это можно сделать 3 способами:
  1. Поставить перед пробелом \\, тогда следующий символ будет считан к предыдущему слову
  2. Обременить последовательность символов в одинарные кавычки: 'aba aba' == aba\ aba
  3. Обременить последовательность символов в двойные кавычки: "aba aba" == aba\ aba
- Файлы/директории вида: "-l", "-la.txt", "-tututu-sahur.jpg"
будут считаться как параметры
